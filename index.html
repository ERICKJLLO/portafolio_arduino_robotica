¿Qué es Arduino?

Arduino es una plataforma de hardware y software libre que facilita la creación de proyectos electrónicos y de programación. Combina placas sencillas, sensores y actuadores con un entorno de programación accesible; es ideal para aprender haciendo.

Breve historia y aplicaciones

Desde su aparición, Arduino ha sido usado en educación, prototipado y arte interactivo. Ejemplos comunes: sistemas de iluminación controlada, robots sencillos, sensores ambientales y proyectos IoT. En este curso verás ejemplos prácticos que van desde encender un LED hasta leer sensores y comunicarlos por Serial.

¿A quién va dirigido?

  • Principiantes sin experiencia previa en electrónica o programación.
  • Profesores que quieren material didáctico y prácticas guiadas.
  • Estudiantes y aficionadas/os que desean experimentar rápidamente con hardware.

Requisitos y material

  • Ordenador con navegador y conexión a Internet.
  • Opcional: placa Arduino (UNO/Nano) y cable USB para pruebas en hardware real.
  • Componentes básicos por proyecto: LED, resistencias, cables, protoboard; cada proyecto incluye la lista concreta.

Qué aprenderás

  • Instalación y uso del IDE de Arduino y del editor en línea.
  • Estructura básica de un programa: setup() y loop().
  • Lectura y escritura digital/analógica, PWM, y manejo de tiempos (delay vs millis).
  • Depuración con Serial y buenas prácticas de montaje y seguridad.
  • Pruebas en simulador antes de volcar código a la placa física.

Consejos de aprendizaje

  • Comienza por proyectos simples y avanza incrementalmente.
  • Modifica los tiempos y valores del código para ver efectos inmediatos.
  • Usa la consola Serial para entender el flujo del programa.

Seguridad básica

Apaga la alimentación antes de cambiar conexiones, respeta polaridad en LEDs y sensores, y usa resistencias cuando sea necesario. Si trabajas con motores o dispositivos que consumen mucha corriente, utiliza fuentes externas y protección adecuada.

Instalación del IDE de Arduino

Pasos y recomendaciones para instalar y preparar el entorno de trabajo, tanto en ordenador como para usar el editor en línea.

1) Elegir el editor

  • IDE clásico (descargable): instala desde https://www.arduino.cc/en/software. Permite trabajar offline y gestionar bibliotecas localmente.
  • Arduino Web Editor: funciona en el navegador y guarda el código en la nube; útil si no quieres instalar nada.

2) Notas por sistema operativo

  • Windows: puede pedir controladores (drivers). Muchas placas chinas usan el chip CH340; instala su driver si Windows no reconoce la placa.
  • macOS: normalmente no requiere drivers para placas oficiales, pero puede pedir permisos de acceso al puerto serie (otorga permiso en Preferencias del Sistema si aparece).
  • Linux: añade tu usuario al grupo dialout o equivalente para permitir acceso al puerto serie: sudo usermod -a -G dialout $USER (luego reinicia sesión).

3) Conectar y seleccionar la placa

  1. Conecta la placa por USB.
  2. En el IDE: Herramientas → Placa → selecciona la placa (Ej. Arduino Uno).
  3. Herramientas → Puerto → elige el puerto COM / /dev/tty que corresponde.

4) Probar con el ejemplo Blink

Abre Archivo → Ejemplos → 01.Basics → Blink y pulsa Subir. Si falla, revisa el puerto, cable y drivers.

5) Monitor Serial y depuración

Usa Herramientas → Monitor Serial para ver mensajes con Serial.print(). Ajusta la velocidad (baud) con Serial.begin(9600) en setup().

6) Bibliotecas

Si un proyecto necesita librerías adicionales (Ej. sensores, displays), instálalas desde Herramientas → Administrar bibliotecas. Lee la documentación de cada librería para ejemplos.

7) Problemas comunes

  • IDE no encuentra el puerto: prueba otro cable USB, otro puerto del PC o reinstala drivers.
  • Error de compilación por librería: instala la versión requerida o revisa el README de la librería.
  • Subida fallida: cierra programas que usan el puerto serie (p. ej. otro monitor) y vuelve a intentarlo.

Cómo programar: conceptos y ejemplos

Aquí ampliamos conceptos clave y ejemplos prácticos para que puedas entender y modificar código con confianza.

Estructura básica

setup() — se ejecuta una vez: configurar pines, iniciar Serial. loop() — se ejecuta repetidamente: lógica principal del programa.

Entradas y salidas

  • pinMode(pin, MODE): define el modo (INPUT, INPUT_PULLUP, OUTPUT).
  • digitalRead(pin) / digitalWrite(pin, value): leer y escribir señales digitales (HIGH/LOW).
  • analogRead(pin): lectura analógica (0..1023).
  • analogWrite(pin, value): PWM (0..255) en pines compatibles para simular niveles intermedios.

Ejemplo: lectura de botón con debounce

// const int BTN = 2;
const int LED = 13;
unsigned long lastChange = 0;
const unsigned long debounce = 40;
int state = HIGH;
void setup(){ pinMode(BTN, INPUT_PULLUP); pinMode(LED, OUTPUT); }
void loop(){
  int reading = digitalRead(BTN);
  if(reading != state && (millis() - lastChange) > debounce){
    state = reading;
    lastChange = millis();
    digitalWrite(LED, state == LOW ? HIGH : LOW); // botón activo a LOW con INPUT_PULLUP
  }
}

Temporización: delay vs millis

delay() detiene el microcontrolador y no permite otras tareas. Con millis() puedes gestionar múltiples temporizadores y mantener la respuesta.

Debug con Serial

Usa Serial.begin(9600) en setup() y Serial.println() para imprimir valores y entender el comportamiento en tiempo real.

// void setup(){ Serial.begin(9600); }
// void loop(){ Serial.println(analogRead(A0)); delay(500); }

Buenas prácticas

  • Usa constantes descriptivas para pines.
  • Evita código duplicado y separa en funciones.
  • Prefiere millis() para tareas periódicas y mantén el loop() ligero.
  • Documenta el código con comentarios y pasos de prueba.
